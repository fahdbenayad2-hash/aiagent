- TypeScript + Node.js (`node:sqlite`, `fetch()`), pas de Playwright
- Credentials dans `.env` uniquement, pas dans le code ; GitHub Secrets pour CI
- Snapshot JSON phase 0 (`output/snapshot-YYYY-MM-DD.json`) continue de fonctionner
- Tous les champs de `/get/notifications` (18-19) sont mappés
- DB path hors repo pour survivre à `git clean -ffdx`
- `GROQ_API_KEY` optionnel : run continue sans crash, analyses de fallback produites
- Si Groq échoue, ne pas crasher le run — log erreur, fallback
- Prompts IA en arabe (Darija), `response_format: { type: "json_object" }`, `Authorization: Bearer`

## Key Decisions
- `better-sqlite3` → `node:sqlite` (Node 22+, zero dep)
- Gemini → Groq (quota épuisé) ; `Bearer` header, pas query param
- Prompts arabes (Darija) pour Groq
- `response_format: { type: "json_object" }` force JSON valide
- Query string DataTables copiée depuis DevTools pour chaque endpoint
- Suivi en batch (`TrackingsList[]`) au lieu de N requêtes séquentielles
- Erreur collecte suspendus/non-encaisse ne crashe pas le run
- Login POST avec `Accept: application/json` + `X-Requested-With: XMLHttpRequest` → Laravel retourne 204 (JSON Auth success) au lieu de 302
- CSRF token extrait depuis la page warmup `/livraison/cashOut` (session authentifiée), pas depuis `/login`
- Collecteurs envoient `X-CSRF-TOKEN` header (token brut) plutôt que `X-XSRF-TOKEN` (cookie crypté)
